# klp-design-system — overview

## What this is

A React 19 + TypeScript + Vite design system built on **Radix Primitives** (behavior + a11y) and **Tailwind v4** (styling via `--klp-*` CSS variables). Distribution model is **shadcn-style copy-paste** via the `klp-ui` CLI: consumers own the source of each component they install.

## Architecture

### 3-layer token system

Tokens flow from raw values to Tailwind utilities through three CSS files under `src/styles/tokens/`:

1. **`primitives.css`** — raw palettes + scales (`--klp-color-*`, `--klp-spacing-*`, `--klp-radius-*`). Internal — components must never reference primitives directly.
2. **`aliases.css`** — semantic tokens (`--klp-bg-*`, `--klp-fg-*`, `--klp-border-*`, `--klp-size-*`, `--klp-font-*`) switched by `[data-brand]`. Components consume these.
3. **`theme.css`** — Tailwind v4 `@theme inline` block exposing **aliases only** as `klp-*` utilities. Powers the `bg-klp-*`, `text-klp-*`, `border-klp-*`, `rounded-klp-*`, `font-klp-*` class families.

The orchestrator `src/styles/tokens.css` imports the three layers and reserves a `--- manual additions ---` marker for project-specific extras (currently a `--font-klp-mono` stack).

### 4 brands, runtime-switched

`[data-brand="wireframe|klub|atlas|showup"]` on the document root selects which alias values are active. No `[data-theme="dark"]`, no `prefers-color-scheme`. See [`brands/_index_brands.md`](./brands/_index_brands.md).

### Per-anatomy-layer cva

Components split styles into **one cva block per anatomy layer** (root / label / icon-left / icon-right). This guarantees a token like `--klp-fg-on-emphasis` lands on the correct DOM node (the label span / icon span, never the root) and prevents `tailwind-merge` from collapsing different `text-*` utilities — `cn()` is configured with extended classGroups so `text-klp-fg-*` (color) and `text-klp-text-*` (font-size) coexist.

## Component pipeline

`/klp-build-component <FigmaNodeName>` orchestrates the pipeline in four stages:

| Stage | Agent | What it does |
|---|---|---|
| 1 | [`figma-extractor`](../.claude/agents/figma-extractor.md) | Captures the Figma component into `.klp/figma-refs/<name>/spec.json` with per-layer token bindings + reference screenshots. Records `captureBrand` (informational — for playground activation). |
| 2 | [`component-adapter`](../.claude/agents/component-adapter.md) | Generates the React source, playground route, and registry stub. Uses literal token-to-utility mapping, no guessing. |
| 3 | [`klp-token-validator`](../.claude/skills/klp-token-validator/SKILL.md) | Skill — runs `scripts/validate-tokens.mjs`. Asserts each layer × state × property in the source uses the correct Tailwind alias utility. Deterministic, <1s, single inline retry on mismatches. |
| 4 | [`documentalist`](../.claude/agents/documentalist.md) | Generates `docs/components/_index_<name>.md`, updates `klp-components.json`, runs the cross-reference reverse-index pass. Non-blocking. |

## How to add a component

Open Figma, pair the Claude Console plugin, select the component, then in Claude Code:

```
/klp-build-component <FigmaNodeName>
```

That's it. The pipeline takes care of extraction, code generation, visual validation, and documentation.

## How to consume from a downstream app

The `klp-ui` CLI reads `registry/index.json` and copies a component's source files into the consumer's repo (shadcn pattern). Each component's registry entry lives at `registry/<name>.json`. The consumer never installs `@klp/ui` as a runtime dependency — they own the source.

## Source-of-truth hierarchy

Every fact in the system has exactly one canonical source. When sources disagree, the higher one wins:

1. **Figma file** `[NEW] UI Kit Klepierre` (file key `khboDX9DqkNBG2gcJnSQTa`)
2. **`.klp/tokens.json`** — captured from Figma variables, regenerated by `pnpm sync:tokens`
3. **`src/styles/tokens/{primitives,aliases,theme}.css`** — generated from tokens.json
4. **`.klp/figma-refs/<name>/spec.json`** — captured per component via figma-extractor
5. **`src/components/<name>/<PascalName>.tsx`** — generated from spec by component-adapter
6. **`docs/components/_index_<name>.md`** — generated from spec + source by documentalist
